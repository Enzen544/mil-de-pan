
## 6. Resumen para cada requisito del enunciado

- Clase abstracta general → `EntidadBase`
- Clase Cliente con validaciones/encapsulación → `cliente.py`
- Clase abstracta Servicio + 3 hijas con polimorfismo → `servicio.py`
- Clase Reserva con estado, confirmación, cancelación → `reserva.py`
- Métodos sobrecargados → `calcular_costo(impuesto=, descuento=)`
- Excepciones personalizadas → `excepciones.py`
- try/except, try/except/else, try/except/finally, encadenamiento → `reserva.py`, `main.py`
- Archivo de logs → `logger_config.py` + `logs/eventos.log`
- Mínimo 10 operaciones simuladas → `main.py` (tiene 13)
- Sin base de datos (listas/objetos) → confirmado en todo el proyecto
